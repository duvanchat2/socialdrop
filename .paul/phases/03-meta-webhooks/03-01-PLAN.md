<objective>
## Goal
Create the Meta Webhooks NestJS module that handles GET verification and POST event reception for Instagram/Facebook webhooks.

## Purpose
This is the entry point for all automation — Meta sends events here (comments, DMs, postbacks). The webhook service identifies the event type and emits it for the flow engine (Phase 04) to consume.

```gherkin

When Meta sends GET /api/webhooks/meta?hub.mode=subscribe&hub.verify_token=<token>&hub.challenge=<challenge>

And returns HTTP 403 when hub.verify_token does not match
```

```gherkin
Then the endpoint returns HTTP 200 immediately
And webhook.service parses the payload and logs the event type (comment | message | postback)
```

<task type="auto">
  <name>Task 1: Create webhook.service.ts</name>
  <files>apps/api/src/modules/webhooks/webhook.service.ts</files>
  <action>
    Create Injectable service with two methods:
    
    verifyWebhook(mode: string, token: string, challenge: string): string | null
    - If mode === 'subscribe' AND token === process.env.META_WEBHOOK_VERIFY_TOKEN
      return challenge
    - Otherwise return null
    
    processEvent(payload: any): void
    - Extract payload.object (should be 'instagram' or 'page')
    - Iterate payload.entry[], then entry.messaging[] for DMs, entry.changes[] for comments/postbacks
    - For each event, identify type:
      * messaging[] with message.text → type = 'message'
      * messaging[] with postback → type = 'postback'  
      * changes[] with field='comments' → type = 'comment'
    - Log each event: Logger.log(`[webhook] ${type} from ${senderId}`, 'WebhookService')
    - Emit EventEmitter2 event 'webhook.event' with { type, senderId, payload: rawEvent }
      (use EventEmitter2 from @nestjs/event-emitter — check if it's in package.json first;
       if not present, just use Logger.log only and add a TODO comment for Phase 04)
    
    Use Logger from @nestjs/common (same pattern as posts.service.ts).
    Import ConfigService from @nestjs/config to read META_WEBHOOK_VERIFY_TOKEN.
  </action>
  <verify>npx nx build api --skip-nx-cache (no TS errors in webhook.service.ts)</verify>
  <done>AC-2 partially satisfied: service parses and logs events</done>
</task>

<task type="auto">
  <name>Task 2: Create webhook.controller.ts</name>
  <files>apps/api/src/modules/webhooks/webhook.controller.ts</files>
  <action>
    Create @Controller('webhooks') with ApiTags('webhooks').
    
    GET /webhooks/meta — verifyMeta(@Query() query):
    - Extract hub.mode, hub.verify_token, hub.challenge from query
    - Call webhookService.verifyWebhook(mode, token, challenge)
    - If result: return result (plain string — add @Header('Content-Type', 'text/plain'))
    - If null: throw ForbiddenException
    
    POST /webhooks/meta — receiveMeta(@Body() body, @Res({ passthrough: true }) res):
    - Set status 200 immediately (Meta requires fast 200)
    - Call webhookService.processEvent(body) in background (do not await if async)
    - Return { status: 'ok' }
    
    Use decorators: @Get('meta'), @Post('meta'), @HttpCode(HttpStatus.OK)
    Import ForbiddenException, Header, Res from @nestjs/common.
    Do NOT add auth guards — Meta webhooks must be public endpoints.
  </action>
  <verify>npx nx build api --skip-nx-cache (no TS errors in webhook.controller.ts)</verify>
  <done>AC-1 satisfied: GET verifies token, returns challenge or 403; AC-2 satisfied: POST returns 200</done>
