# PR #19 — 🔴 Validar firma X-Hub-Signature-256 de webhooks Meta

**Área:** Seguridad
**Prioridad:** 🔴 Crítico
**Esfuerzo estimado:** S (< 2h)

**Problema:**
`WebhookController.receiveMeta` procesa el body **sin verificar la firma HMAC** de Meta. Cualquiera que conozca la URL puede inyectar eventos falsos y disparar flows/DMs automáticos en nombre del sistema. Solo el GET de verificación (verify token) está implementado.

**Solución propuesta:**
Capturar el **raw body** (configurar `rawBody: true` en Nest para esa ruta) y calcular `HMAC-SHA256(FACEBOOK_APP_SECRET, rawBody)`; comparar con la cabecera `X-Hub-Signature-256` usando `crypto.timingSafeEqual`. Rechazar con 401 si no coincide o falta.

**Archivos a modificar:**
- `apps/api/src/modules/webhooks/webhook.controller.ts`
- `apps/api/src/main.ts` (rawBody para `/api/webhooks/meta`)

**Criterios de aceptación:**
- [ ] Petición sin firma válida → 401 (no se procesa el evento)
- [ ] Comparación en tiempo constante
- [ ] Test con firma válida/ inválida / ausente
