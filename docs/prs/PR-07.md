# PR #07 — Simplificar acciones de "Nuevo Post" y navegación

**Área:** UX
**Prioridad:** 🟢 Mejora
**Esfuerzo estimado:** M (2-8h)

**Problema:**
Cuatro botones con semántica solapada en `/posts/new` ("Añadir a la Cola" / "Publicar ahora" / "Programar" / "Guardar borrador") y 18 links en la navegación (11 main + 4 automatización + 3 config). Carga cognitiva alta. Además no se puede escribir caption sin subir media primero.

**Solución propuesta:**
1. Colapsar a **un botón primario contextual** ("Añadir a la cola" si hay slots; "Publicar/Programar" según haya fecha) + menú "⋯" con acciones secundarias (borrador).
2. Agrupar navegación: mover `Debug Logs` a footer/rol admin; agrupar Analytics/Competencia/Mi Voz bajo "Insights".
3. Permitir caption **sin** media (post solo-texto para Twitter/FB); validar en `commonGuards` que IG/TikTok/YouTube sí exijan media.

**Archivos a modificar:**
- `apps/web/src/app/posts/new/page.tsx`
- `apps/web/src/components/AppShell.tsx`

**Criterios de aceptación:**
- [ ] Máximo 2 acciones visibles al crear post
- [ ] Se puede crear post de solo texto donde la plataforma lo permita
