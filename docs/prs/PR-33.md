# PR #33 — Sistema de tokens de diseño (color, tipografía, radios)

**Área:** Diseño / Frontend
**Prioridad:** 🟡 Importante (habilita PR #34-#37; no bloquea el lanzamiento funcional)
**Esfuerzo estimado:** M (2-8h)

**Problema:**
No existe sistema de diseño: `apps/web/src/app/global.css` es solo un reset con Tailwind v4 (`@import "tailwindcss"`) **sin un solo token propio** (`@theme` vacío), la fuente es la del sistema (`ui-sans-serif`), y todos los colores son utilidades `gray-800/900` + `indigo-*` hardcodeadas y dispersas por componente. Cualquier rediseño coherente exige una fuente de verdad única.

**Solución propuesta:**
Definir el sistema completo en `global.css` vía `@theme` (Tailwind v4 CSS-first — no hace falta `tailwind.config`):

```css
@theme {
  /* Superficies — elevación por color, NO por borde ni sombra (máx 3 niveles) */
  --color-base:      #0A0A0F;  /* página: negro real */
  --color-surface:   #17171F;  /* card, sin borde visible */
  --color-surface-2: #1F1F2A;  /* elevado / hover / celdas activas */
  /* Tinta */
  --color-ink:       #F2F2F7;  /* primario */
  --color-ink-muted: #8E8FA3;  /* secundario/labels */
  /* Acentos semánticos (texto/íconos; SIEMPRE con ícono o signo, nunca solo color) */
  --color-accent:    #6366F1;  /* violeta de marca — se conserva el actual del sidebar */
  --color-positive:  #A3E635;  /* lima: deltas ▲, publicado */
  --color-warning:   #FBBF24;  /* ámbar: deltas ▼, errores, alertas */
  /* Series de gráficas — orden FIJO, validado CVD/contraste sobre #17171F */
  --color-chart-1: #7C7AF5;  --color-chart-2: #3FA34D;
  --color-chart-3: #C77F00;  --color-chart-4: #3A9BDC;
  /* Radios */
  --radius-card: 16px;  --radius-chip: 10px;  --radius-pill: 999px;
  /* Motion (consumidos por PR #34) */
  --duration-hover: 175ms;  --duration-enter: 350ms;  --ease-out: cubic-bezier(0.16,1,0.3,1);
}
```

**Tipografía** (vía `next/font` en `layout.tsx`, self-hosted, `display: swap`):
- **Display** `Space Grotesk` (next/font/google, 500/700): H1 28px/1.2 bold · títulos de card 20px/1.3 semibold.
- **Body** `Satoshi` (next/font/local, woff2 de Fontshare — licencia gratuita; fallback aceptado: `Switzer`, misma fuente de origen): body 14px/1.5 · small 12.5px · label 11px uppercase tracking `0.08em`.
- **Mono** `JetBrains Mono` (next/font/google, 400/600) **reservada para cifras**: KPIs 28-32px semibold `tabular-nums`, contadores/timestamps/IDs 12px. Regla dura: todo número de datos (no de prosa) va en mono con `font-variant-numeric: tabular-nums`.

**Reglas transversales que fija este PR (heredadas del sistema de dataviz, aplican a #36/#37):**
1. Elevación solo por color de superficie; los bordes desaparecen (divisores de tabla: `rgba(255,255,255,0.04)`).
2. Máximo 3 acentos activos por vista (violeta + lima + ámbar); el 4º dato pasa a tinta muted.
3. Estados nunca color-solo: cada badge lleva ícono + label (✓ publicado-lima, ○ programado-violeta, ⚠ error-ámbar, ─ borrador-muted).
4. Los textos/labels de gráficas usan tokens de tinta, nunca el color de la serie; las series usan `--color-chart-*` en orden fijo, sin ciclar.
5. Focus visible: anillo 2px `--color-accent` con offset 2px (al quitar bordes, el focus no puede depender de border).
6. **Dark-first:** `html` arranca con `.dark`; el toggle claro/oscuro del `ThemeProvider` se oculta tras flag `NEXT_PUBLIC_ENABLE_LIGHT_THEME` (light queda deprecated en esta fase — decisión reversible, el provider no se elimina).

**Archivos a modificar:**
- `apps/web/src/app/global.css` (tokens `@theme` + reglas base)
- `apps/web/src/app/layout.tsx` (next/font: Space Grotesk, Satoshi local, JetBrains Mono; clase dark por defecto)
- `apps/web/public/fonts/` (woff2 de Satoshi)
- `apps/web/src/components/ThemeProvider.tsx` (flag del toggle)

**Criterios de aceptación:**
- [ ] Todos los tokens viven en un solo `@theme`; ningún hex nuevo hardcodeado en componentes a partir de este PR
- [ ] Las 3 fuentes cargan self-hosted vía next/font (cero layout shift, `swap`)
- [ ] La paleta de series pasa `validate_palette.js` sobre `#17171F` (ya validado: 4/4 checks)
- [ ] Contraste: `--color-ink` ≥ 7:1 y `--color-ink-muted` ≥ 4.5:1 sobre `--color-surface`
- [ ] Dark por defecto; toggle oculto tras flag sin romper el ThemeProvider

**Notas técnicas:**
En Tailwind v4 los tokens `@theme` generan las utilidades (`bg-surface`, `text-ink-muted`, `rounded-card`, `font-mono-nums` vía utility custom). El lima/ámbar de UI (#A3E635/#FBBF24) son deliberadamente más brillantes que los pasos de serie (#3FA34D/#C77F00): los primeros son texto (WCAG ≥4.5:1), los segundos marcas de gráfica (banda de luminosidad dark validada) — no intercambiarlos.
