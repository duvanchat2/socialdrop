# PR #11 — Paginación real en listados de posts

**Área:** Performance
**Prioridad:** 🟢 Mejora
**Esfuerzo estimado:** M (2-8h)

**Problema:**
El calendario llama `/api/posts?userId=…` **sin rango** y trae todos los posts con `include` profundo (integraciones + media); crece sin techo con el uso.

**Solución propuesta:**
1. El calendario debe pedir por **rango de mes** usando `getCalendar(from,to)` (ya existe en `PostsService`).
2. Añadir cursor/paginación a `findAll`.
3. Usar `select` con solo los campos que la UI de pills necesita.

**Archivos a modificar:**
- `apps/web/src/app/calendar/page.tsx`
- `apps/api/src/modules/posts/posts.service.ts`
- `apps/api/src/modules/posts/posts.controller.ts`

**Criterios de aceptación:**
- [ ] El calendario solo carga el rango visible (query key por mes)
- [ ] Payload por post reducido a lo que la UI usa
