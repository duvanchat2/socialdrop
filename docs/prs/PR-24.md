# PR #24 — Error tracking (Sentry/GlitchTip) backend + frontend

**Área:** Observabilidad
**Prioridad:** 🟡 Importante
**Esfuerzo estimado:** M (2-8h)

**Problema:**
No hay captura estructurada de excepciones. Los errores de React (render, promesas sin catch) se pierden en el navegador del usuario; los 500 del backend solo quedan en logs del contenedor.

**Solución propuesta:**
Integrar Sentry (SaaS) o GlitchTip (self-hosted en el VPS si se prefiere no depender de terceros). Backend: `@sentry/nestjs` con filtro global de excepciones. Frontend: `@sentry/nextjs` + `app/error.tsx` y `app/global-error.tsx` (hoy no existen). Priorizar eventos: fallos de publicación, errores de Graph API, excepciones no capturadas, 500s.

**Archivos a modificar:**
- `apps/api/src/main.ts` (+ filtro global)
- `apps/web/src/app/layout.tsx`, nuevos `apps/web/src/app/error.tsx` y `global-error.tsx`

**Criterios de aceptación:**
- [ ] Excepciones backend y frontend llegan al tracker con contexto (userId, ruta)
- [ ] Existe ErrorBoundary global en el frontend
