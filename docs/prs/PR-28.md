# PR #28 — Límites de uso por plan (cuotas Apify/IA)

**Área:** Feature / Infraestructura
**Prioridad:** 🟢 Mejora (post-Stripe)
**Esfuerzo estimado:** M (2-8h)

**Problema:**
Los costos variables por usuario no tienen techo: análisis de competidores (descarga de video + transcripción faster-whisper en CPU del VPS + Claude por post) y generación de guiones escalan linealmente con el uso. Sin cuotas no se pueden definir planes rentables.

**Solución propuesta:**
1. Contadores por usuario/mes en Redis + tabla `UsageCounter { userId, metric, period, count }`: competidores analizados, guiones IA generados, syncs manuales.
2. Middleware/guard de cuota en los endpoints con costo (`competitors/analyze`, `brain/generate`, `assistant`).
3. Al exceder el plan: bloquear con mensaje claro o degradar (ej. análisis sin IA, solo transcript).
4. Exponer consumo del período en la UI (settings o dashboard).

**Archivos a modificar:**
- Nuevo `apps/api/src/modules/usage/*`
- `apps/api/src/modules/competitors/*`, `apps/api/src/modules/brain/*`
- `libs/prisma/prisma/schema.prisma` (`UsageCounter`)

**Criterios de aceptación:**
- [ ] Cada acción con costo verifica cuota antes de ejecutar
- [ ] Consumo visible por usuario
- [ ] Límites configurables por plan
