# PR #25 — Persistir Debug Logs por usuario (no efímero global)

**Área:** Observabilidad
**Prioridad:** 🟢 Mejora
**Esfuerzo estimado:** S (< 2h)

**Problema:**
`DebugLogService` guarda logs en una lista Redis de 500 entradas keyed a `demo-user` → se pierden con rotación, se mezclarían entre usuarios en multi-tenant, y los eventos importantes (publicación fallida) no sobreviven.

**Solución propuesta:**
1. Keyear por `userId` real (tras PR #12).
2. Para eventos críticos (publicación fallida, needsReauth), persistir además en una tabla `EventLog { id, userId, level, context, message, error?, createdAt }` con retención (job de limpieza a 30 días).
3. Redis queda solo para el stream en vivo del panel `/debug`.

**Archivos a modificar:**
- `apps/api/src/modules/debug/debug-log.service.ts`
- `libs/prisma/prisma/schema.prisma` (`EventLog` + migración)

**Criterios de aceptación:**
- [ ] Logs separados por usuario
- [ ] Eventos críticos persistidos con retención configurable
