# PR #23 — Dead-letter queue + alertas de fallo (Telegram/email)

**Área:** Observabilidad
**Prioridad:** 🟡 Importante
**Esfuerzo estimado:** M (2-8h)

**Problema:**
Los jobs fallidos solo se logean en `@OnWorkerEvent('failed')`; una publicación que agota reintentos muere en silencio. No existe ningún canal de alertas (email/Slack/Telegram) cuando falla una publicación o se cae una integración.

**Solución propuesta:**
1. Cola `dead-letter`: cuando `attemptsMade >= attempts`, mover el job y persistir el motivo.
2. Nuevo `AlertService` con canal Telegram Bot (barato y simple) y/o email (Resend). Disparar en: publicación agota reintentos, integración pasa a `needsReauth` (PR #21), health check en rojo.
3. Panel mínimo en `/debug` para ver la dead-letter y reencolar.

**Archivos a modificar:**
- `apps/api/src/modules/posts/processors/post-scheduler.processor.ts`
- Nuevo `apps/api/src/modules/alerts/*`
- `apps/web/src/app/debug/page.tsx` (sección dead-letter)

**Criterios de aceptación:**
- [ ] Job que agota reintentos genera alerta y queda en dead-letter
- [ ] Alertas configurables por canal (env)
- [ ] Se puede reencolar desde la UI de debug
