# PR #05 — Fix responsive: tablas y grids que desbordan en móvil

**Área:** Mobile
**Prioridad:** 🟡 Importante
**Esfuerzo estimado:** M (2-8h)

**Problema:**
`competitors/page.tsx` y el dashboard (`page.tsx`) renderizan `<table>` sin contenedor scrollable → en <768px la página entera hace scroll horizontal. El calendario mensual (`grid-cols-7`, `min-h-[92px]`) queda ilegible en móvil.

**Solución propuesta:**
1. Envolver toda `<table>` en `<div className="overflow-x-auto -mx-4 px-4">`.
2. Dashboard: en móvil convertir filas en tarjetas apiladas o mantener tabla con scroll propio.
3. Calendario: por debajo de `sm`, usar la vista lista (`ListView` ya existe) por defecto.
4. Adoptar breakpoints consistentes: `sm=640` móvil grande, `md=768` tablet, `lg=1024` desktop (sidebar fija), `xl=1280` contenedores `max-w-6xl`. Regla: contenido tabular siempre dentro de `overflow-x-auto`; nunca anchos fijos en px para texto.

**Archivos a modificar:**
- `apps/web/src/app/competitors/page.tsx`
- `apps/web/src/app/page.tsx`
- `apps/web/src/app/calendar/page.tsx`

**Criterios de aceptación:**
- [ ] Ninguna página produce scroll horizontal del `body` a 360px
- [ ] Toda tabla ancha scrollea dentro de su propio contenedor
- [ ] Calendario usable en móvil (lista o agenda)
